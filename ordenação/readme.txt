Bubble Sort: Este algoritmo é simples e fácil de implementar, mas não é muito eficiente para grandes conjuntos de dados. 
Ele se destaca quando você tem um pequeno conjunto de dados ou um conjunto de dados que já está quase ordenado.

Merge Sort: Este algoritmo é eficiente para grandes conjuntos de dados, pois tem uma complexidade de tempo O(n log n) em todos os casos. 
Ele se destaca quando você precisa de um algoritmo de ordenação estável que funcione bem em grandes conjuntos de dados.

Quick Sort: Este algoritmo é geralmente mais rápido do que o Merge Sort para conjuntos de dados menores, 
mas sua velocidade pode diminuir para conjuntos de dados maiores ou para conjuntos de dados que já estão ordenados. 
Ele se destaca quando você tem um conjunto de dados de tamanho moderado que está aleatoriamente ordenado.
